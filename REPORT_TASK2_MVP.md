# Отчет по Заданию 2: MVP проекта

## Описание проекта

ReviewMind — AI-агрегатор обзоров товаров. Telegram-бот принимает название продукта или ссылки на обзоры, собирает данные из YouTube, Reddit, 4PDA и веб-страниц, сохраняет их в векторную базу и формирует структурированный ответ на основе RAG-пайплайна с использованием gpt-4o-mini.

Два режима работы:
- Auto — бот ищет источники самостоятельно (YouTube, Reddit, 4PDA, экспертные порталы)
- Manual — пользователь присылает ссылки, бот анализирует именно их


## Принятые решения и обоснование стека

**Qdrant вместо pgvector / MongoDB Atlas Vector Search.** Qdrant — специализированная векторная БД с нативной payload-фильтрацией и score boost. Для системы, где ANN-поиск является core-операцией, это правильный инструмент. PostgreSQL остается для реляционных данных (пользователи, логи, источники).

**Две коллекции Qdrant (curated_kb + auto_crawled).** Разные жизненные циклы и уровни доверия. Curated контент постоянный и верифицированный, auto_crawled может быть stale или biased. Разделение позволяет бустить curated без изменения алгоритма поиска.

**gpt-4o-mini вместо reasoning-моделей (o4-mini).** Задача — синтез текста и суммаризация, не многошаговые логические рассуждения. gpt-4o-mini надежно следует структурированным промптам при стоимости ~10x ниже.

**Celery + Redis вместо FastAPI BackgroundTasks.** BackgroundTasks работают in-process и не переживают рестарт. Celery-задачи durable, retryable, мониторятся через Flower и масштабируются горизонтально. Критично при скрапинге, занимающем 3-5 минут.

**aiogram 3.x.** Лучшая async Telegram-библиотека для Python. Нативная поддержка middleware, FSM, inline keyboards.

**trafilatura для веб-скрапинга.** Извлекает чистый текст статьи из любого URL без headless-браузера. Достаточно для MVP.

**Long polling вместо webhook.** Long polling не требует домена, HTTPS, Nginx. Для разработки и раннего тестирования запускается мгновенно. Переход на webhook при появлении Web UI.

## Сложности и их решение

**1. YouTube transcript extraction блокируется без cookies.** youtube-transcript-api возвращает ошибки для возрастного контента. Решение: поддержка cookies.txt через конфиг `YOUTUBE_COOKIES_PATH`. Позволяет обходить ограничения без headless-браузера.

**2. aiogram F.text.regexp не матчит URLs в manual mode.** Фильтр `F.text.regexp(URL_PATTERN)` не срабатывал. Причина: aiogram проверяет полное совпадение, а не поиск подстроки. Решение: замена на `re.search` в хендлере (коммит `6e38fe8`).

**3. Rate limiting в Telegram API.** Бот получает 429 при массовой рассылке. Решение: slowapi с лимитом 10 req/min per user_id, admin bypass.

**4. Cosine dedup при upsert в Qdrant.** Без дедупликации один и тот же чанк сохранялся многократно. Решение: перед upsert проверяем cosine similarity > 0.95 — если найден дубликат, пропускаем. Детерминистические UUID на основе контента.

**5. Sponsor detection для RU/EN контента.** 26 regex-паттернов для обнаружения спонсорских вставок на двух языках. Используется для downweight (x0.7) при ранжировании.

**6. Sync PRAW в async-контексте.** Reddit API (PRAW) синхронный, а весь проект async. Решение: вызовы PRAW через `asyncio.to_thread()`. В логах предупреждение о переходе на asyncpraw, но для MVP текущее решение достаточно.

## Использование LLM

**Инструменты:** GitHub Copilot (Claude Sonnet 4 / Claude Opus 4), ChatGPT (gpt-4o).

**Стоимость:**
- GitHub Copilot Pro — $39/мес (подписка, используется для всего проекта)
- OpenAI API (разработка + тесты) — ~$0 (gpt-4o-mini бесплатный по подписке Github по небольшому лимиту, text-embedding-3-small еще дешевле)
- GitHub Student Pack — бесплатный доступ к GitHub Models для dev-среды
- Итого: ~$39

**Где LLM ускорил работу:** генерация boilerplate (модели, миграции, тесты), написание regex-паттернов для sponsor detection, отладка Docker Compose конфигурации, создание Mermaid-диаграмм для README.

**Где LLM не помог:** архитектурные решения (две коллекции Qdrant, порядок стадий разработки) принимались вручную на основе опыта. Отладка runtime-ошибок при деплое (SSH порты, proxy routing) требовала ручного анализа логов.
